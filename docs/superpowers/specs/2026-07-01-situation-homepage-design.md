# 设计文档：航空装备质量安全态势 — 首页态势大屏（Phase 1）

- **日期**：2026-07-01
- **状态**：待评审（Draft for review）
- **目标项目**：`quality-safety-situation`（jeecgboot-vue2 架构：`ant-design-vue-jeecg` 前端 + `jeecg-boot` 后端）
- **设计依据**：`设计图首页.png`（完全复刻目标，本 Phase 唯一真值源）、`航空装备质量安全态势-原型.html`（后续 Phase 功能参考）

---

## 1. 目标与范围

### 1.1 目标
在现有 jeecgboot-vue2 工程内，新增一个**全屏"首页态势大屏"**，对 `设计图首页.png` 做**像素级复刻**，作为整个系统的入口与总览。大屏上的对象可下钻进入 jeecgboot 管理后台做明细处置。

### 1.2 多阶段分解（本项目体量大，按阶段切片）
原型涵盖安全打分、案例推理、贝叶斯真因、八维画像、备件、战区/基地等多个子系统，**远超单个 spec**。按用户既定优先级切片：

| 阶段 | 内容 | 状态 |
|------|------|------|
| **Phase 1（本 spec）** | 首页态势大屏的完全复刻（视觉 + Mock 交互） | 本文档 |
| Phase 2+ | 大屏下钻的管理后台模块、其余导航页、真实后端数据 | 各自独立 spec |

### 1.3 Phase 1 范围

**纳入（IN）**
- 首页大屏的静态像素复刻：Header + 左/中/右三列全部面板。
- 所有面板使用内置 **Mock 数据**渲染，图表可交互（中国地图、进度条/仪表、风险清单等）。
- Header 实时时钟走时。
- 顶部导航 Tab、地图底部子 Tab 可点击切换；非首页 Tab 用"建设中"占位。
- **1920×1080 基准 + 等比缩放**适配其它分辨率。
- 大屏可点击对象 → 下钻跳转到 jeecgboot 后台路由（目标路由 Phase 1 可为空壳页）。

**不纳入（OUT，属 Phase 2+）**
- 真实后端数据与 DB 表结构、真实接口。
- 其余导航页（画像中心 / 专题分析 / 分析工作台 / 指标与模型 / 全链路留痕）的完整业务内容。
- 真实 AI 叙事生成、真实预测/置信度模型。
- 后台各管理模块（机队/机型/故障/质量/备件等）的 CRUD 页面。

## 2. 架构与集成

### 2.1 技术栈
- **前端**：保持 jeecgboot 的 **Vue 2.6 + Ant Design Vue 1.7.2**，不引入 React/新框架。
- **图表**：新增 **ECharts 5**（`echarts` 主包 + Vue2 薄封装 `BaseChart.vue`，封装 `init/resize/dispose`）。
- **大屏装饰**：科技边框面板用**纯 CSS/Less 手写**（`.tech-panel` 角标 + 标题栏），不引入 DataV 类组件库。

### 2.2 路由与入口（对应"大屏入口 → 下钻后台"）
- 在 `src/router/index.js` 的 **constantRoutes** 新增静态路由 **`/situation`**，**不套 admin 布局**（无顶栏/侧边栏，全视口，类似登录页处理）。
- 登录成功后默认重定向到 `/situation`（大屏作为系统落地页）。
- 大屏可点击对象（机队 / 机型 / 故障 / 风险项 / 地图节点 等）→ 跳转 `/sys/...`、`/quality/...` 等 jeecgboot 后台路由。

### 2.3 分辨率适配
- 设计稿基准 **1920×1080**。
- 大屏根容器外层用 **`transform: scale()` 等比缩放**铺满视口（依据 `window.innerWidth/Height` 与 1920×1080 的比值取较小者，居中），保留比例不拉伸。
- 监听 `resize` 重新计算；辅以 `rem` 做字号微调。封装为 `useScale.js`（mixin 或工具）。

### 2.4 数据层（对应"Mock 数据，预留接口"）
- 新建 `src/views/situation/mock/`，**每个面板一个 Mock 文件**，字段形状 = 未来后端 API 的返回结构。
- 统一 `src/views/situation/api/situationApi.js`，函数如 `getQualityIndicators()` 内部先 `return mock`，注释标明对应真实接口（如 `GET /situation/quality-indicators`）。
- Phase 2 后端就绪时**只改这一层**，组件不动。

## 4. 组件拆分与文件结构

```
src/views/situation/
  index.vue                       # 大屏主页：scale 壳 + Header + 三列网格
  components/
    BaseChart.vue                 # ECharts 薄封装（init/resize/dispose + 统一主题）
    TechPanel.vue                 # 可复用科技边框面板（标题栏+角标+tab槽+slot）
    MetricBar.vue                 # 可复用指标条（左列质量安全、右列任务能力各用5次）
    RiskListItem.vue              # 可复用风险/预测条目（已识别风险、预测风险共用）
    SituationHeader.vue           # 标题+导航Tab+下拉+预警按钮+KPI条+安全口径
    CenterMap.vue                 # 中国地图+底部子Tab+右下小图
    DisposalProgress.vue          # 处置措施进展
    NarrativeBox.vue              # AI态势解读叙事线
  modules/
    LeftColumn.vue                # 左列：质量安全指标 + 已识别风险
    RightColumn.vue               # 右列：任务能力 + 处置 + 叙事 + 预测
  mock/
    header.js  qualityIndicators.js  riskList.js  centerMap.js
    taskIndicators.js  disposal.js  narrative.js  prediction.js
  api/
    situationApi.js               # 函数返回 mock，注释标明真实接口；Phase2 仅改此层
  theme/
    echarts-theme.json            # 统一图表主题
    variables.less                # 颜色/字体 token
  situation.less                  # 大屏级样式 + .tech-panel
  useScale.js                     # 1920×1080 等比缩放适配
```

路由：`src/router/index.js` → constantRoutes 增 `path: '/situation'` → `views/situation/index.vue`（无 admin 布局）。

## 5. 样式系统
- `variables.less`：导出全部颜色 token（见 3.1）+ 字体栈。
- `.tech-panel` 类：面板背景 + `#1e4a7a` 描边 + 4px 圆角 + 四角科技角标（CSS 伪元素）+ 顶部标题栏（含 tab 切换槽）。
- `echarts-theme.json`：统一坐标轴/网格/文字/配色，所有 `BaseChart` 默认载入，保证图表风格一致。
- 大号数字用 Rajdhani 字体类（`.num-tech`）。

---

## 6. 交互与下钻行为
- 顶部导航 Tab：点击切换高亮；非"总体态势"页内容区显示"建设中"占位（Phase 1）。
- 地图底部子 Tab：点击切换高亮 + 切换地图 Mock 高亮数据集。
- 地图节点 / 风险条目 / 指标项：hover 高亮；点击 → `$router.push` 到对应后台路由（Phase 1 目标可为空壳页）。
- Header 时钟：每秒更新（`setInterval`，组件销毁时清除）。
- `总体态势预警` 按钮：点击触发一个 Mock 预警提示（Ant Design Vue `message`/弹窗），Phase 1 仅交互壳。

## 7. 验收标准（Phase 1）
1. 浏览器 1920×1080 下，大屏与 `设计图首页.png` **视觉一致**（版面、配色、面板、图表类型、中文标签）。
2. 控制台无报错；所有 ECharts 图表正常渲染并随窗口 resize 自适应。
3. 其它分辨率（如 1366×768、2K、4K）等比缩放，不变形、不出现滚动条。
4. 顶部 Tab / 地图子 Tab 可切换；Header 时钟走时。
5. 下钻对象点击可跳转到后台路由（不报错）。
6. 现有工程 `npm run lint` 与 `npm run build` 通过。
7. Mock 数据层结构清晰，`situationApi.js` 注释标明每个函数对应的未来真实接口。

## 8. 风险与待确认
- **R1 ⚠️ 文案精度**：导航 Tab（态势改版等）、地图子 Tab（中部保障区/北部保障基地/无键命中）、AI 叙事细节文字为 OCR 读出，需对照设计图原件在实现时核对修正。
- **R2 登录落地策略**：默认登录后跳 `/situation`——需确认是否覆盖 jeecgboot 默认落地页（默认通常是 `/dashboard` 或 `/`），避免与既有行为冲突。
- **R3 下钻路由**：Phase 1 下钻目标路由尚未定义，先以空壳页承接，Phase 2 再补真实模块。
- **R4 字体许可**：Rajdhani 经 Google Fonts 加载（原型已用），需确认部署环境可联网加载或改为本地字体文件。
